Guia do Projeto - Visão por Computador

Este guia descreve a organização do projeto por etapas, incluindo tarefas, decisões técnicas e justificações para cada abordagem adotada.

Etapa 1 — Estudo do vídeo e Segmentação

Tarefa 1 — Estudo do vídeo

Visualizar várias frames do vídeo para identificar características relevantes: cor das laranjas, fundo, reflexos e objetos semelhantes.

A análise inicial é essencial para evitar escolhas erradas mais tarde. Sem compreender o vídeo, os thresholds serão arbitrários.
Definir que o problema deve ser resolvido com base em cor (e não apenas intensidade).

Tarefa 2 — Teste de canais RGB

Utilizar funções de extração de canais RGB para observar diferenças entre vermelho, verde e azul.

Os canais RGB ajudam a perceber se existe separação natural entre objeto e fundo, mas raramente são suficientes em cenários com iluminação variável.
Conclusão de que RGB não é robusto para este problema.

Tarefa 3 — Conversão para HSV

Converter a imagem RGB para HSV utilizando a função apropriada.

HSV separa cor (Hue) de intensidade (Value), tornando a segmentação mais robusta em presença de reflexos.
Escolha do espaço de cor HSV como base da solução.

Tarefa 4 — Segmentação HSV

Aplicar segmentação com intervalos de Hue, Saturation e Value ajustados ao tom das laranjas.

O parâmetro mais importante é o Hue. Saturation ajuda a eliminar o fundo branco, e Value remove regiões muito escuras.
Imagem binária onde as laranjas estão destacadas.

Tarefa 5 — Operações morfológicas

Aplicar abertura e fecho para remover ruído e preencher falhas.

Abertura remove pequenos pontos isolados. Fecho corrige falhas dentro dos objetos.
Máscara limpa e pronta para análise.

Etapa 2 — Histogramas, Equalização e Blobs

Tarefa 6 — Histogramas

Gerar histogramas para analisar a distribuição de intensidades ou cores.

Os histogramas permitem justificar a escolha dos thresholds em vez de usar tentativa e erro.
Escolha fundamentada de thresholds.

Tarefa 7 — Equalização

Testar equalização de histograma para melhorar contraste.

Nem sempre melhora o resultado. Em imagens coloridas, pode distorcer a informação cromática.
Decisão de usar ou descartar equalização (normalmente descartar).

Tarefa 8 — Conversão para 1 canal

Converter a máscara RGB binária para imagem de um canal.

As funções de blob labelling requerem imagens binárias de um canal.
Imagem pronta para etiquetagem.

Tarefa 9 — Etiquetagem de blobs

Identificar objetos individuais na imagem.

Cada blob corresponde a um objeto candidato a laranja.
Lista de blobs identificados.

Tarefa 10 — Extração de propriedades

Obter área, perímetro, bounding box e centro de gravidade.

Estas métricas são essenciais para filtrar objetos e classificá-los.
Dados geométricos completos por objeto.

Etapa 3 — Geometria, Calibre e Categoria

Tarefa 11 — Filtragem de objetos

Eliminar objetos que não correspondem a laranjas.

Usar critérios como área, forma e proporções evita falsos positivos.
Lista de laranjas válidas.

Tarefa 12 — Circularidade

Calcular circularidade para avaliar a forma.

Valores próximos de 1 indicam formas circulares, características das laranjas.
Métrica de forma para cada objeto.

Tarefa 13 — Diâmetro equivalente

Calcular o diâmetro com base na área.

Este método é mais robusto do que usar apenas largura ou altura.
Diâmetro em píxeis.

Tarefa 14 — Conversão para milímetros

Converter píxeis para unidades reais.

A escala fornecida no enunciado deve ser usada diretamente.
Diâmetro real.

Tarefa 15 — Calibre

Classificar o tamanho da laranja segundo normas.

A classificação deve seguir intervalos definidos, não valores arbitrários.
Código de calibre atribuído.

Tarefa 16 — Categoria

Determinar qualidade da laranja (Extra, I, II).

Basear em forma, uniformidade e defeitos visuais. É uma aproximação computacional.
Categoria atribuída.

Etapa 4 — Integração Final e Tracking

Tarefa 17 — Pipeline completo

Integrar todas as etapas num fluxo contínuo.

O pipeline deve ser organizado e sequencial para facilitar debugging.
Sistema funcional.

Tarefa 18 — Contagem na frame

Contar objetos válidos por frame.

Número de laranjas atual.

Tarefa 19 — Tracking

Associar objetos entre frames.

Evita contar a mesma laranja várias vezes.
Identificação consistente dos objetos.

Tarefa 20 — Contagem total

Manter contagem acumulada.

Número total de laranjas.

Tarefa 21 — Visualização

Desenhar informação sobre o vídeo.

A visualização deve ser clara e não sobrecarregar a imagem.
Interface final com dados.

Tarefa 22 — Testes finais

Executar e ajustar parâmetros.

A afinação é essencial para reduzir erros.
Sistema validado.

Conclusão

O projeto segue uma abordagem estruturada baseada em segmentação por cor, análise morfológica, extração de características e classificação. As decisões técnicas foram fundamentadas em testes experimentais e observação do comportamento do sistema.
